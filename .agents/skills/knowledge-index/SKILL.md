# Knowledge Index

Thin entrypoint. The canonical, verifiable procedure lives in the Harness skill
**`_harness/skills/generate-knowledge-index.md`** — read and follow it. This
file only adapts that procedure for an agent's "skill" surface (e.g. Codex
`/skills`).

The index is split into a deterministic part (regenerated by the CLI) and an
authored part (preserved between HTML-comment markers). Do **not** hand-edit the
deterministic sections.

## Steps

1. **Scaffold** the deterministic sections (idempotent; preserves authored
   content):

   ```bash
   _harness/bin/harness-cli knowledge scaffold
   ```

2. **Author** the semantic parts in `docs/KNOWLEDGE_INDEX.md`:
   - `Purpose` — between `KNOWLEDGE:PURPOSE:BEGIN/END` (1–3 sentences).
   - `Key Concepts` — between `KNOWLEDGE:CONCEPTS:BEGIN/END`; reference
     `_harness/docs/GLOSSARY.md`, don't duplicate it.
   - Replace each `TODO: describe.` in **Top-Level Structure** with a one-line
     description.

3. **Format** (repo uses Prettier `proseWrap: always`):

   ```bash
   npx prettier --write docs/KNOWLEDGE_INDEX.md
   ```

4. **Verify** — the mechanical gate. Must exit `0`:

   ```bash
   _harness/bin/harness-cli knowledge check
   ```

## Done when

`harness-cli knowledge check` is green, Prettier is clean, Purpose and Key
Concepts are authored, and every Top-Level Structure entry has a description.
